# Product

## Register

product

## Users

Frequent power users — people running this repeatedly (event staff, HR/school admins doing recurring batch print jobs), not one-off casual visitors. They move through Design → Data → Preview → Print on a regular cadence and value speed and predictability over hand-holding.

## Product Purpose

A browser-based tool to design ID card templates, map CSV data onto them, capture or upload photos, and print individual, selected, or all cards — with no backend, everything local. Success looks like: a repeat user can get from a template + CSV to printed cards with minimal friction and no surprises in output fidelity.

## Brand Personality

Fast & utilitarian. Lean, dense, efficient — minimal chrome, no flourish that slows down repeat use. Feels like serious print-production tooling for someone who uses it often, not a showcase or a first-time-friendly onboarding experience.

## Anti-references

Generic AI-SaaS dashboard look: purple-to-blue gradients, hero-metric cards, nested cards, bloated onboarding flows, tiny uppercase eyebrows, numbered section scaffolding. This is a workflow tool, not a landing page or a consumer app.

## Design Principles

- Speed over hand-holding — favor dense, efficient layouts for repeat users over guided, spacious first-run experiences.
- Workflow-first structure — the four-step flow (Design, Data, Preview, Print) is the backbone; UI should reinforce where the user is in that sequence, not bury it.
- Print-fidelity is non-negotiable — anything shown on screen (card canvas, preview grid) must represent what actually prints; no decorative deviation from true layout/scale.
- No decorative flourish at the cost of density — utilitarian precision beats visual polish that adds friction for a power user running the same flow repeatedly.

## Accessibility & Inclusion

Standard WCAG AA baseline (contrast, keyboard navigation, reduced-motion support). Not a focused priority for this project at this stage — no additional constraints beyond the baseline.
